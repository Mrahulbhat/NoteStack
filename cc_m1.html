Q1: What is Cloud Computing? Explain the vision of Cloud Computing with diagram.

Cloud computing is a technology that allows users to access and use computing resources, such as servers, storage, databases, and applications, over the internet. Rather than owning and maintaining physical hardware or software, users rent these resources from cloud service providers on-demand.

  • Flexibility: Users can rent resources as needed without paying upfront costs.
  • Scalability: Resources like computing power, storage, and networking can be easily increased or decreased based on demand.
  • Cost-effectiveness: Users only pay for the resources they use, which reduces unnecessary costs.
  • Efficiency: Cloud computing enables faster setup, development, and management of applications and services.

Vision of Cloud Computing:

  • On-demand Access: Users can quickly rent virtual resources and manage them online without paying upfront.
  • Utility-based Model: In the future, cloud services could be traded like utilities, where services are bought and sold as needed.
  • Support for Developers: Developers can rent storage or computing power to run their applications without owning infrastructure.
  • Green IT: Optimizes data centers, saves energy, and reduces carbon emissions, contributing to a greener ecosystem.

Q2: How is Cloud Computing helping enterprises, governments, and institutions?

Cloud computing is transforming how various sectors manage computing resources by providing on-demand, scalable, and cost-effective services. It helps:

  • Large Enterprises: They offload data storage and application hosting to cloud systems, reducing costs and improving efficiency.
  • Small Enterprises & Start-ups: They can quickly bring ideas to life without large upfront costs using the "pay-as-you-go" model.
  • System Developers: Developers can focus on business logic while the cloud handles infrastructure complexity, servers, and scalability.
  • End Users: They can access documents and applications from any device, enhancing flexibility and convenience.

Q3: Explain the Cloud Computing Reference Model with a diagram.

Cloud computing offers three main types of services:

  • Infrastructure as a Service (IaaS): Provides virtual hardware (servers, storage, networking) on demand. Users can run their software and applications on virtual machines.
  • Platform as a Service (PaaS): Offers developers a platform to build, run, and manage applications without dealing with the underlying infrastructure.
  • Software as a Service (SaaS): Delivers ready-to-use applications to users via the internet. No need for installation, e.g., Google Docs.

Summary: IaaS provides infrastructure control, PaaS supports application development, and SaaS delivers ready-to-use software.

Q4: Define Distributed Systems. List and Explain the 3 major milestones of distributed systems that led to the evolution of Cloud Computing.

Ans: A distributed system is a collection of independent computers that appears to users as a single, unified system. These computers work together to perform tasks, share resources, and process information, but they operate independently and are located in different places.

The 3 Major Milestones leading to cloud computing are:

  1. Mainframe Computing:

    These were the first examples of large computational systems. Mainframes were powerful, highly reliable computers specialized for large data movement and massive input/output (I/O) operations. They were mostly used by large organizations for bulk data processing tasks such as online transactions, enterprise resource planning, and other operations involving the processing of significant amounts of data.

  2. Cluster Computing (1980s):

    What It Is: Cluster computing came as a cheaper alternative to mainframes and supercomputers. It connects many smaller, inexpensive computers (called a cluster) through a fast network, allowing them to work together as one powerful system.

    Contribution: Cluster computing allowed tasks to be processed faster and more efficiently by sharing the workload across many machines. It introduced the idea of using multiple computers together, which is a key concept in cloud computing.

  3. Grid Computing:

    Grid computing built on cluster computing by linking computers across different locations—sometimes across countries or even the world. This created access to much greater computing power and storage, making it possible to solve more complex problems.

    Contribution: Grid computing introduced the idea of sharing resources on a large scale and using them only when needed. This idea is central to modern cloud computing, where resources like storage and computing power can be accessed on demand.

    Key Developments:

    • Clusters became common: Many organizations started using clusters, so the technology became widely available.
    • Underutilized clusters: Often, these clusters weren’t fully used, leaving room for improvement.
    • Need for more power: Certain problems needed more computational power than a single cluster could handle, driving the need for larger grids.
    • Better networking and Internet: Advances in network technology and the rise of the Internet made it possible to connect computers over long distances with fast, high-bandwidth connections.

Q9: Explain Web 2.0 and its significance in cloud computing.

The web is a key way that cloud computing provides its services. Today, the web includes various technologies and services that enable interactive information sharing, collaboration, user-centered design, and application building. This change is known as Web 2.0, which represents a new way for developers to create applications and deliver services online, improving the experience for users.

Key Features of Web 2.0:

  • Interactivity and Flexibility: Web 2.0 adds interactivity to web pages, allowing users to access features typically found in desktop applications.
  • Technologies: This is made possible by combining standards and technologies like XML, AJAX (Asynchronous JavaScript and XML), and web services. These tools enable developers to create applications that rely on user input, turning users into content creators.
  • Dynamic Applications: Web 2.0 applications are very dynamic—they continuously improve and regularly integrate new updates and features based on how the community uses them. There’s no need to install new software on users’ devices.
  • Accessibility: These applications aim to reach a wide range of internet users by being accessible in terms of media and cost.

Examples of Web 2.0 Applications:

  • Google Docs
  • Google Maps
  • Flickr
  • Facebook
  • Twitter
  • YouTube
  • Delicious
  • Blogger
  • Wikipedia

Social networking sites, in particular, benefit greatly from Web 2.0. The level of interaction seen on sites like Facebook and Flickr relies on technologies like AJAX and RSS (Really Simple Syndication), which make the user experience much more engaging.

The idea of the web as a tool that enhances interaction was introduced by Darcy DiNucci in 1999 and became fully realized around 2004. Today, it serves as a strong foundation for cloud computing, which heavily relies on Web 2.0. Applications and frameworks that provide rich internet applications (RIAs) are essential for making cloud services accessible to everyone.

Q6: Outline any six characteristics of cloud computing.

Cloud computing offers several key characteristics that provide advantages to both cloud service consumers (CSCs) and cloud service providers (CSPs):

  1. No Up-Front Commitments:

    Users don’t need to buy expensive hardware or software in advance. They can start using resources when needed and pay as they go.

  2. On-Demand Access:

    Users can access resources (like storage, processing power, and applications) whenever they need them, without waiting for setup or delivery.

  3. Flexible Pricing:

    Users only pay for what they use, making it a cost-effective option compared to maintaining in-house infrastructure.

  4. Simplified Application Scalability:

    Applications can be scaled up or down easily based on demand, allowing for fast and efficient adjustments without needing to manage infrastructure.

  5. Efficient Resource Allocation:

    Cloud systems automatically allocate resources based on demand, ensuring that they are used efficiently and without waste.

  6. Energy Efficiency:

    By sharing resources across multiple users, cloud computing reduces energy consumption, leading to more eco-friendly operations.

  7. Seamless Integration of Third-Party Services:

    Cloud platforms allow easy creation and integration of services from third-party providers, enabling users to quickly extend their system with new features and tools.

Q7: Explain Service-oriented computing's two important concepts which are fundamental to cloud computing.

Service-oriented computing (SOC) is a foundational model for cloud computing. In this approach, services are the main building blocks used to develop applications and systems.

Key Features of SOC:

  • It allows for the rapid development of low-cost, flexible, and interoperable systems.
  • A service is a self-contained component that can perform various functions, from simple tasks to complex business processes.
  • Services are designed to be:
    • Loosely Coupled: They can work independently, making them adaptable and reusable in different situations.
    • Reusable: Services can be used in multiple applications, improving efficiency.
    • Programming Language Independent: Services can be accessed by clients using different programming languages.
    • Location Transparent: Clients can use services from anywhere, without knowing their physical location.

Two Important Concepts of SOC:

  1. Quality of Service (QoS):

    QoS refers to a set of attributes that measure how well a service performs. This includes:

    • Performance (e.g., response time)
    • Security
    • Reliability
    • Scalability
    • Availability
  2. Software-as-a-Service (SaaS):

    SaaS is a delivery model where software applications are provided over the internet. Instead of buying and installing software, users can access it on a subscription or rental basis from a central data center. This concept was inspired by Application Service Providers (ASPs), which made software available via the web.

Q8: List and explain any 4 Cloud computing platforms and technologies.

1. Amazon Web Services (AWS):

AWS is a leading cloud service provider offering a comprehensive range of Infrastructure-as-a-Service (IaaS) solutions.

Key Features:
  • Elastic Compute Cloud (EC2): Provides customizable virtual servers that enable users to deploy scalable applications quickly. Users can select from various configurations, including options for GPU and cluster instances.
  • Simple Storage Service (S3): Offers scalable and durable storage solutions, allowing users to store and retrieve any amount of data. S3 organizes data into "buckets," making it easy to manage and access objects from anywhere.
  • Image Creation: Users can save running instances as images, allowing for easy replication and deployment of applications.

2. Google App Engine:

Google App Engine is a scalable platform for developing and hosting web applications, leveraging Google’s robust infrastructure.

Key Features:
  • Automatic Scaling: Dynamically adjusts resources to meet user demand, ensuring applications perform optimally under varying loads.
  • Development Environment: Provides a Software Development Kit (SDK) that allows developers to build and test applications locally before deployment.
  • Supported Languages: Supports multiple programming languages, including Python, Java, and Go, along with built-in services like in-memory caching and data storage.

3. Microsoft Azure:

Azure is a versatile cloud platform that acts as an operating system for cloud-based applications and services.

Key Features:
  • Role-Based Architecture: Applications are organized into roles—Web roles for hosting applications, Worker roles for processing tasks, and Virtual Machine roles for customizable environments.
  • Diverse Services: Offers a wide range of services for different application needs, from hosting websites to managing complex workloads.
  • Integration: Seamlessly integrates with Microsoft products, enhancing productivity for businesses already using Microsoft solutions.

4. Force.com and Salesforce.com:

Force.com is a cloud platform designed for building social enterprise applications, serving as the backbone for Salesforce.com, a leading customer relationship management (CRM) solution.

Key Features:
  • Component-Based Development: Enables developers to build applications using pre-configured components, streamlining the development process.
  • Comprehensive Development Support: Provides tools for every aspect of application design, including data modeling, business logic, workflows, and user interface creation.
  • Enterprise Solutions: Tailored to support the operational needs of enterprises, enhancing collaboration and efficiency across teams.

These platforms empower businesses to innovate and operate efficiently by providing scalable, flexible, and cost-effective solutions tailored to various needs.
